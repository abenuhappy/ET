## ⚠️ 확인 필요 사항

### 1. data.json 공개 여부
- `data.json`에는 지출 내역과 거래처 정보가 포함되어 있습니다
- **공개 저장소에 푸시하면 모든 사람이 볼 수 있습니다**
- **선택지**:
  - ✅ **공개해도 괜찮다면**: 그대로 푸시
  - 🔒 **비공개로 하고 싶다면**: 
    - `.gitignore`에 `data.json` 추가
    - 또는 비공개 저장소(private repository) 사용

### 2. 기본 비밀번호
- `app.py`에 기본 비밀번호 `jinipini0608`가 하드코딩되어 있습니다
- **하지만** 환경 변수 `APP_PASSWORD`로 오버라이드 가능합니다
- **권장**: Render에서 `APP_PASSWORD` 환경 변수를 설정하여 변경

### 3. SECRET_KEY
- 기본값 `dev-secret-key-change-in-production`이 있습니다
- **필수**: Render에서 `SECRET_KEY` 환경 변수를 반드시 설정하세요

## 📋 푸시 전 최종 체크리스트

```bash
# 1. 변경사항 확인
git status

# 2. 커밋할 파일 확인
# 필수 포함:
# - app.py (JSON 기반으로 수정됨)
# - data.json (GitHub 자동 로드를 위해 필요)
# - export_to_csv.py (JSON 기반으로 수정됨)
# - migrate_to_json.py (새 파일)
# - RENDER_SETUP.md (새 파일)
# - README.md (업데이트됨)
# - build.sh (업데이트됨)

# 3. 커밋 (선택사항: expenses_export.csv는 제외 가능)
git add app.py data.json export_to_csv.py migrate_to_json.py RENDER_SETUP.md README.md build.sh
git commit -m "Migrate from SQLite to JSON file storage with GitHub auto-load"

# 4. 푸시
git push
```

## 🚀 Render 배포 시 환경 변수 설정

Render 대시보드 → Environment 탭에서 설정:

**필수:**
```
SECRET_KEY=<랜덤 문자열 생성>
APP_PASSWORD=<원하는 비밀번호>
```

**GitHub 자동 로드 (권장):**
```
GITHUB_REPO=username/repo-name
GITHUB_BRANCH=main
```

## 💡 팁

- `data.json`을 공개하고 싶지 않다면, 별도의 비공개 저장소를 만들어서 `GITHUB_REPO`에 설정할 수 있습니다
- 또는 `.gitignore`에 `data.json`을 추가하고, 수동으로 Render에 업로드하는 방법도 있습니다
